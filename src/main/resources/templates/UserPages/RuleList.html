CONSIDERACIONES
  • ETHER TYPE: Elige si la regla debe asociarse al protocolo IPv4 o IPv6.
  • TCP/UDP: Usa los campos de puerto. Puedes definir un único puerto o un rango.
  • ICMP/ALL: Los campos de puerto no se usan.
  • Remote CIDR: Especifica el rango de IPs remotas permitidas.
  • Dirección: Elige si la regla es para tráfico entrante o saliente.
¿Cómo funcionan las reglas de un Security Group?

Las reglas definen la seguridad de tus VMs

Una regla define qué tráfico está permitido para las interfaces/ports asociadas a este Security Group.
Por defecto, todo el tráfico está bloqueado, excepto ARP y conexiones tanto establecidas como relacionadas (se utiliza "connection tracking").

  • Dirección: "ingress" (entrante) o "egress" (saliente)
  • Ether Type: "ipv4" o "ipv6"
  • Protocolo: "tcp", "udp", "icmp", "all"
  • Puertos: Solo para TCP/UDP. Puedes definir un puerto único o un rango. Para ICMP/ALL no se usan puertos.
  • Remote CIDR: Rango de IPs permitidas en formato "A.B.C.D/E". 0.0.0.0/0 para cualquier IP.
  • Descripción: Campo opcional para identificar la regla.
Ejemplos:
Permitir SSH entrante
  • Protocolo: TCP
  • Puerto: 22
  • Dirección: Entrante (ingress)
  • IP CIDR: 10.0.20.8/32 (IP única)
Permitir rango UDP saliente
  • Protocolo: UDP
  • Puertos: 1000-2000
  • Dirección: Saliente (egress)
  • IP CIDR: 10.0.0.0/8 (red interna)
Permitir todo ICMP entrante
  • Protocolo: ICMP
  • Dirección: Entrante (ingress)
  • IP CIDR: 0.0.0.0/0 (cualquier IP)
